# 虚拟机linux环境搭建

# 几种linux环境搭建方式

  1. 购买一台主机，安装linux系统，成本太贵
  2. 安装windows，linux双系统，切换麻烦
  3. 云服务器linux系统，本质也是虚拟化技术，太贵，容量小
  4. windows系统的wsl2方案，不能多节点，不稳定
  5. 虚拟机linux系统，成本低，可部署多节点，适合学习使用
  6. Docker技术

# 准备工作

# 安装VMware虚拟机

windows虚拟机有Vmware Workstation Player和Vmware Workstation Pro两个版本，前者是免费版，已经足够使用了

Mac系统有Vmware Fusion

# 准备linux系统镜像文件

这里准备的是CentOS-7-x86_64-DVD-1708.iso

可以在

有ubuntu、centos、debain各种linux系统，不同版本号的镜像

# 宿主机设置

  1. 点击控制面板-程序和功能-启用或关闭Windows功能
  2. 勾选虚拟机平台
  3. 勾选Windows虚拟机监控程序平台
  4. 重启宿主机

# 安装Linux系统

新建虚拟机：

  1. VM选择创建虚拟机，选择驱动程序位置
  2. 固件类型选择传统BIOS
  3. 配置处理器核数、硬盘容量、内存，本机上设置1核、内存1G、硬盘20G
  4. 选择拆分硬盘，（不拆分的话会直接蓝屏）
  5. 网络模式选择NAT

启动虚拟机：

  1. 选择语言为简中
  2. 软件选择：按照需求去选择
  3. 设置用户名密码

# 网络配置

# 桥接模式和NAT模式

桥接模式：虚拟机就像是局域网的一台独立主机，可以访问网络内的任何机器，主机网卡和虚拟网卡IP地址处于同一个网段，子网掩码、网关、DNS等参数都相同。

NAT模式（网络地址转换）：虚拟机获得私有IP地址，而宿主机充当路由器，将虚拟机的网络流量转发到外部网络。

注：

  1. 桥接模式适用于虚拟机需要和外部网络进行通信
  2. NAT模式适用于需要在虚拟化环境对网络流量进行控制管理

# 桥接模式配置

1.打开终端，查看网卡信息

ifconfig

这时虚拟机的ens33网卡是没有IP地址的，

注：ens33是较新的系统中的第一个以太网接口,老版本是eth0、eth1的命名方式

2.打开VM软件，找到对应的虚拟机，右键设置，在网络适配器选项，勾选桥接模式，并且勾选复制物理网络连接状态

注：因为本地使用的宿主机是台笔记本，连接的网络是 WIFI,并且需要经常更换

3.分配IP地址

// 切换到root模式，不然dhclient会报错
su root 

// 分配一个可用的IP地址
dhclient 

// 查看ip地址
ifconfig

4.编辑网卡配置

// 打开网卡配置
vim /etc/sysconfig/network-scripts/ifcfg-ens33

更改网卡下列选项

// 表示网络接口使用静态IP地址，默认是dhcp
BOOTPROTO="static"
// 表示系统启动时自动激活该网络接口
ONBOOT="yes",默认是no
// 配置IP地址，为第3步拿到的IP地址
IPADDR="192.168.137.165"  
// 子网掩码
NETMASK="255.255.255.0"
// 网关
GATEWAY="192.168.137.1"
// DNS，用宿主机的DNS就可以
DNS1="192.168.0.1"

保存退出

// 重启网卡
systemctl restart network.service

5.验证

// 是否联通外网
ping www.baidu.com
// 是否联通宿主机
ping 192.168.137.x
